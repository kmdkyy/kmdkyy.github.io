bash脚本执行提示/bin/sh^M: bad interpreter错误

2021年9月14日 11:49:19

在执行shell脚本出现：

./a.sh
bash: ./a.sh: /bin/bash^M: bad interpreter: No such file or directory

说明是文件格式问题，解决方法：

  1. vim a.sh进入a.sh后， 在底部模式下， 执行:set fileformat=unix后执行:x或者:wq保存修改。 然后就可以执行./a.sh运行脚本了。

  2. 直接执行sed -i "s/\r//" a.sh来转化， 然后就可以执行./a.sh运行脚本了。
